Website architectural requirements define the layout of the website e.g., what services each of the components provide. It also ensures the design meets the business and client requirements.

Main architectural requirements are:

  • Technical constraints e.g., Interfaces for sever, storage, communication, and memory.
  • Functional aspects include website services and processes.
  • Visual appearance defines the visual side of the webpage such as the interface, colour, buttons, and other elements.
  • Security parameters ensure the webpage has secure access control and transactions.

Website design structure provides an overview of how to organise the information on the website in a way that provides users with intuitive content access. It is used to illustrate the architecture and the design of the webpage.

Design hierarchy

  • Hierarchical Model
    This design is similar to a tree that branches out into categories and pages. It the most common design structure used in web applications.
  • Sequential Model
    Used to create flows for a process in step-by-step order e.g., when a user is creating an account and has to enter their details.
  • Matrix Model
    Provides options and the user can choose where they want to go next. Matrix model is non-traditional, unique, and the oldest design structure that are navigated through internal links or search.
  • Database Model
    This design structure is dynamic, and it integrates database with search. Database model is based on bottom-up approach that delivers content based on information architecture principles.
Navigation Design

  • Primary Navigation
    Located at the top the page and it groups and holds the bulk of the page structure which helps the users navigate where to click next.
  • Top Bar Navigation
    Positioned above the primary navigation bar, providing easy access to links e.g., log-in.
  • Footer Navigation
    Contains information related to website policies, terms and conditions and other miscellaneous links and it is found at the bottom of the page.
  • Sidebar Navigation
    Used in web pages that have a lot of information. It organises and groups together related topics, making it easier to move between the pages.

The design has to be as simple as possible and provide optimal on-screen experience e.g., accessing webpage on different devices. The production processes have to be logical, clear, and consistent to ensure it meets the user and/or business requirements.

Programming controls structures specify in which way a program flows based on certain parameters or conditions.

  • Sequence aka sequential flow executes the instructions in a sequential order.
  • Iteration aka repetitive flow continues the execution of instructions until a certain condition has been met.
  • Selection aka conditional flow provides alternate paths and only executes a program based on the evaluation of the condition.

  • Functional Testing is a quality assurance (QA) process where testing is done in the source code, and it determines if the program meets the user requirements and specifications.
  • Usability Testing combines both functionality and overall user experience testing. This process uses internal or external testers to examine all functions of the application. The results are then analysed which enables the developers determine if the application needs to be improved.
  • Interface Testing is used to test if the application displays and provides appropriate communication correctly. In addition, it checks if the interactions and interruptions between the web server and application server instances are handled appropriately.
  • Compatibility Testing ensures the application is compatible and functional with all devices e.g., the browser, the operating system, and mobile.
  • Performance Testing checks the resilience of the application by testing it under different conditions e.g., different internet speeds to ensure that it recovers from crashes in the best way possible.
  • Security Testing ensures the application meets all security standards and protects the application from unauthorised access, viruses, and other malicious software.

Website debugging methods help developers to evaluate the functionality of a webpage by inspecting and analysing the accessibility of the web page. If an issue is detected it can be modified by using the different developer tools available online.

Front-end (client-side) technologies are used to develop the interactive components of the website that users see and interact with.
The front-end coding techniques are:

  • Hyper Text Markup Language aka HTML It is the fundamental technique used to create a webpage. It provides the structure and the content of a web application.
  • Cascading Style Sheet aka CSS Provides the style and aesthetics of a web page. It defines the appearance of the webpage by specifying the layout, fonts, and element positioning.
  • JavaScript is a scripting language that gives the website its behaviour and functionality, making it interactive for the user.
